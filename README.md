# 📘 TIL 작성 가이드

## 1. 기록 원칙

1. 그날 배운 **모든 사항**을 **당일** 작성한다.
2. **몰랐던 것**은 무조건 작성한다.

## 2. 작성 방식

- **개조식** 중심으로 간결하게 작성 (한눈에 보기 쉽게)
- **마크다운 문법**을 사용하여 정리
- 예시나 활용 사례는 가독성을 위해 포함
- **절대 복잡하거나 지저분하게 작성하지 않는다**

## 3. GitHub 업로드 규칙

1. `TIL` 전용 레포지토리에 매일 업로드
2. 가능하면 매일 작성 → **1일 1커밋**
3. 업로드가 지연된 경우 **날짜 명시** (예: 작성일 4/21, 업로드일 4/23)
